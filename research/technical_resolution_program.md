# B1-B10 Technical Resolution Program v0.1

Last updated: 2026-06-22

Purpose: keep the portfolio focused on solving the ten technical problems first.
Publication, patent, financing, and product outcomes are downstream consequences
of technical resolution; they are not the current success criterion.

## Current Priority Order

1. **B1 -> B2 -> B7 systems spine.** This is the strongest path because B1
   already has replayable proof evidence, B2 has a real Stim/PyMatching baseline,
   and B7 connects them into resource consequences.
2. **B4 -> B8 verification spine.** This is the second path because it has a
   clear failure boundary: adaptive spoofers become dangerous at high leakage.
3. **B10/B9 theory guardrails.** These prevent us from overclaiming and can
   produce formal negative results.
4. **B3/B5/B6 application spine.** These require stronger classical baselines
   and real data before they can become solution claims.
